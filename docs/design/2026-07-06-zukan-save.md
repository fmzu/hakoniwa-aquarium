# 図鑑＋セーブ 設計ドキュメント

- 日付: 2026-07-06
- 状態: オーナー承認済み
- 目的: 発見した住民の記録（図鑑）と進捗の永続化（セーブ）。M2 の本命機能

## 図鑑のデータ

- 種ごとに記録する: 初発見日時（ISO文字列）・累計誕生数
- 未発見の種は図鑑上で影（シルエット）＋「？」表示。「影の魚を食べると正体が分かる」というゲーム構造の再利用
- 回遊レアの「目撃」記録を将来足せるよう、フィールド拡張可能な形にする

## セーブ（localStorage）

- 保存するもの: version（スキーマ番号・初版は1）+ 図鑑（種→発見日・誕生数）+ 満腹ゲージ
- 保存しないもの: 泳いでいる顔ぶれ・位置（ハンドオフ確定仕様: 顔ぶれは保存せず図鑑から生成）
- タイミング: 誕生・捕食のたびに自動保存。手動セーブなし（セーブし忘れの悲しみを構造的に排除）
- 破損対策: 読み込み時に検証し、不正なら初期状態で開始する（クラッシュさせない）。シリアライズ⇔デシリアライズの往復はユニットテストで保証する
- 注意: localStorage はオリジンに紐づくため、Cloudflare 移行時にセーブは引き継がれない → 移行はセーブ機能で遊び込む前に行う（M2リスト参照）

## 起動時の海（抽選）

- 発見済みの種からランダムに最大3体（種の重複なし）を泳がせて開始する。満腹ゲージはセーブから復元する
- 乱数は注入する（テスト決定性）。将来の滞在スケジューラはこの抽選関数の置き換えで実現する

## 図鑑UI（DOMオーバーレイ）

- 画面下に「ずかん」ボタン。押すとゲームの上に半透明パネルを表示する（開いている間もゲームは進行）
- パネル内: 種ごとのカード（拡大スプライト・名前・発見日・誕生数）。未発見は影＋「？」
- スプライトは既存グリッドデータから小canvasに描画する（画像ファイル不要・`image-rendering: pixelated`）
- 種名は暫定: ラムネ魚 / ストロベリークラゲ / たい焼き（命名規則の本格検討はハンドオフ未決事項のまま）
- 方式の根拠: 文字のドット描画コスト回避・128×90は情報密度が足りない・図鑑は世界でなく額縁（chrome）側・osushi の実績ある方式

## テスト

- 純粋関数（シリアライズ/デシリアライズ往復・検証・抽選・図鑑更新）は bun test（TDD）
- DOM UI はユニットテスト対象外、実機確認で担保する
