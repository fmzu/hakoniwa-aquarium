# 誕生演出 設計ドキュメント

- 日付: 2026-07-05
- 状態: オーナー承認済み
- 目的: 現状プレースホルダ（白リング1個）の誕生演出を本実装する。M2 の最優先項目

## シーケンス（合計約2.5秒・ゲームは止めない）

誕生位置（食べた瞬間の主人公付近、新入りのスポーン位置）で再生:

| 時間 | 演出 |
|---|---|
| 0〜0.3s | 画面全体を均一に暗転（フェードイン） |
| 0〜0.9s | 泡が立ち上る（1pxと2×2ドットの泡、6〜10個） |
| 0.7s | 新入りが影のシルエット（#0F1E27）で出現・静止 |
| 0.9〜1.7s | 頭から尾へ列単位でフレーバー色が染みる。染まり境界の1列は白 #FFFFFF |
| 1.7〜2.3s | 1pxドットのリング拡大＋キャンディ粒（その種のパレット色＋金平糖色 #FFE39A）が飛散。泡はフェードアウト |
| 2.3〜2.5s | 暗転フェードアウト。新入りが通常描画に戻り泳ぎ始める |

## フォーカス（暗転スポットライト）

- 円形マスクは使わない（円の縁がベクター描画になりドット規律違反のため）
- 「全キャンバスを均一に暗くする → その上に新入りと演出を描く」方式。均一処理なので解像度の混在は起きない
- 暗転色は水色系の暗色（例: rgba相当の均一オーバーレイ）。強さは実機で微調整

## ドット規律（このプロジェクトのAD規則に追加）

- エフェクトも1ドット＝論理1px（128×90グリッド）。論理キャンバスへのベクター描画（arc / ellipse / 斜線 stroke 等）は禁止
- 既存違反の捕食小リング（ctx.arc）も本実装でドット化する

## 実装方針

- Resident 型に bornAtMs（誕生時刻）を追加。演出中（elapsed - bornAtMs < 合計時間）は stepResident が移動をスキップし静止。演出後に通常挙動へ
- 純粋関数として切り出しテストする:
  - 経過時間→フェーズ判定と進行度
  - 染まり境界列の計算
  - リングの1pxドット座標列生成（決定的）
  - 泡の軌道（インデックスから決定的に生成、乱数不使用）
- 描画関数: 暗転オーバーレイ / ドットリング（捕食小リングと共用） / 誕生演出の合成描画。draw-scene の描画順を「通常要素 → 暗転 → 新入り＋演出」に拡張
- 数値（時間・泡の数・リング半径等）は定数ファイルに集約し調整可能にする

## スコープ外

- 音
- カメラ移動
- 回遊レア用の特別演出
- 退場演出（色→影の逆再生。滞在スケジューラ実装時にセットで行う）

## 経緯メモ

- 方向性はアニメーションモック3案（影から染まる/泡柱/キャンディスパーク）を比較し「全部の掛け合わせ」に決定
- 描画スタイルはなめらか描画とドット構成を並べて比較し「全部ドット」に決定
- ドットサイズ統一（mixels回避）の指摘を受け、1ドット=論理1pxの規律をエフェクトにも適用することを確認済み
