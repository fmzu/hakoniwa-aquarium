# 今日の顔ぶれ（セッションスケジューラ）設計ドキュメント

- 日付: 2026-07-07
- 状態: オーナー承認済み
- 目的: セッションごとに海の顔ぶれが変わる仕組み。滞在スケジューラの「通い」クラスの箱庭的解釈

## 基本思想（オーナーの言葉）

「今回のセッションではこれだけの数の子たちをずっと表示しようかな〜」のノリ。セッション中の入れ替わりは作らない（数分で去られると寂しい）。減らない・たまに増える・翌日は違う顔ぶれ。

## 仕様

1. **セッション開始時抽選**: 発見済みの種から2〜4体（発見数が少なければその範囲で）を抽選し「今日のメンバー」とする。既存 pickStartingResidents の進化版（抽選数にゆらぎを追加）。決まったメンバーはセッション中ずっと滞在
2. **誕生**: 生まれた子もセッション終了まで滞在
3. **まれな来訪**: ゲーム内時間で約5分に1回抽選し、当たれば発見済み種から1体が画面外（カメラの左右どちらか外）から泳いで登場。定員（RESIDENT_MAX=8）未満のときのみ。来た子も帰らない。頻度・確率は定数で調整可能（0にもできる）
4. **誕生押し出し（唯一の退場）**: 満員（8体）で誕生条件を満たした場合、誕生を優先する。新生児以外で最も早くから居る住民（arrivedAtMs 最小）に退場予定フラグを付け、その住民は通常通り泳ぎ続け、**カメラ外に出た瞬間に静かに消える**（一時的に9体になる期間を許容）。退場演出（影化lerp）は作らない — 将来の回遊クラス用に温存
5. **実時間減衰なし**: 全タイマーはゲーム内 elapsedMs 基準（既存原則の維持）
6. **セーブ**: 変更なし（顔ぶれは保存しない。次回起動時に再抽選）

## 決定ログの更新（docs/spec.md 側）

- 決定3「満員でも満腹は消化する（誕生はしない）」を上書き: 「満員でも誕生する（押し出し方式）。最古参の住民が退場予定となりカメラ外で消える。変更経緯: セッションスケジューラ導入に伴い『食べたのに生まれない』体験を解消（2026-07-07）」

## 実装方針

- Resident に arrivedAtMs（到着時刻）と departing（退場予定フラグ）を追加
- 純粋関数でTDD: 抽選数のゆらぎ / 来訪判定（乱数注入・elapsedMs基準の間隔管理） / 押し出し対象の選定（新生児除く最古参） / カメラ外判定による除去 / 画面外スポーン位置の算出
- stepWorld に組み込み（誕生ブロックの満員分岐を押し出しに変更、退場予定住民のカメラ外チェック）。カメラ位置が必要になるため stepWorld の入力にカメラ情報（または hero位置から導出）を追加する設計判断が必要——計画側で最善の形を検討
- 数値は定数ファイル集約（来訪間隔・確率・抽選数の範囲・保証最低数）

## スコープ外

常駐クラス（プロップとセット）/ 回遊クラス（LL級とセット）/ 影化退場演出 / セッション中の自然退場（作らない）
